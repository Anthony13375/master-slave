*Ответить в свободной форме*

#### Ответ к заданию 1

#### Master-Slave принцип работы:

Существует один главный узел (Master), который принимает все операции записи. Имеется один или несколько подчиненных узлов (Slaves), которые получают изменения от мастера и применяют их у себя. Клиентские приложения обычно пишут данные только на мастер, а читать данные могут как с мастера, так и с слейвов для балансировки нагрузки.

Плюсы:
* Отказоустойчивость чтения
* Масштабирование чтения

Минусы:
* Единая точка отказа записи - мастер
* Задержка репликации на Slave

#### Master-Master принцип работы:

Существует два (или более) мастер-сервера. Каждый из них принимает операции записи. Изменения, сделанные на одном мастере, реплицируются на другой мастер и наоборот. Часто это кольцевая репликация (A->B, B->A). Клиентские приложения могут писать на любой из мастеров.

Плюсы:
* Отказоустойчивость записи
* Масштабирование записи

Минусы:
* Возможны конфликты данных
* Сложность разрешения конфликтов данных
